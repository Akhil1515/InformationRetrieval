The program was written in Python3.6

First import the required packages.
--> From datetime package import datetime which helps to know the execution time of the program
--> From nltk.tokenize import word_tokenize which helps to tokenize the words
--> Import re inorder to use regular expressions
--> Counter from collections is used to count hashable objects
--> PorterStemmer from nltk.stem is used for stemming
--> From nltk.stem.wordnet import WordNetLemmatizer is used for lemmatization

The path to the cranfield location is entered as a command line argument along with the python file "dict.py"

In this program four indexes are being created namely
1. Index_Version_1.uncompressed
2. Index_Version_2.uncompressed
3. Index_Version_1.compressed
4. Index_Version_2.compressed

Constructing Index_Version_1.uncompressed:
------------------------------------------

First, 1400 files are taken and all the sgml tags, punctuations are removed.
Next, tokenization is done using nltk.word_tokenize
Everything is converted into lower case.

Then stopwords are removed.

Lemmas are created using nltk.WordNetLemmatizer()

Inverted index for lemmas will be created by using SPIMI (Single-pass in-memory indexing) and it will be in the following format:

TERM	DOCFREQ		[DOCID,TF,MAX_TF,DOCLEN]

where TERM stands the term
DOCFREQ stands for number of documents in which the term appeared
DOCID represents the document number eg.. 187,1210
TF indicates the number of times the term appeared in the document
MAX_TF is the frequency of the most frequent term or stem in that document
DOCLEN is the total number of word occurrences in the document

The output is written to Index_Version_1.uncompressed
Sample output:

abbott:	    	1	[[1340, 1, 6, 118]]

Constructing Index_Version_2.uncompressed:
------------------------------------------

First, 1400 files are taken and all the sgml tags, punctuations are removed.
Next, tokenization is done using nltk.word_tokenize
Everything is converted into lower case.

Stems are created using nltk.PorterStemmer() method

Inverted index for stems will be created by using SPIMI (Single-pass in-memory indexing) and it will be in the following format:

TERM	DOCFREQ		[DOCID,TF,MAX_TF,DOCLEN]

where TERM stands the term
DOCFREQ stands for number of documents in which the term appeared
DOCID represents the document number eg.. 187,1210

Constructing Index_Version_1.compressed:
------------------------------------------

For dictionary compression a blocked compression with k=8 is used and for the posting file gamma encoding for the gaps between document-ids is used.

Blocked compression:
Combine the entire terms as a single string and insert length of each term before the term
Since k=8 keep an index (namely a pointer) 
This will compress the file and reduce the size
Ex: 4aaaa6ababab2bh

Gamma encoding:
To write 5 in gamma and delta codes we perform the following operations:
1. write 5 in binary as 101
2. For the gamma code remove the leading 1-bit to obtain the offset: 01
3. The length of the offset is 2:
• In unary the length is 110
4. The code of 5 in gamma is 11001 (or 110,01 – to represent [length, offset])
5. For the delta code, the length of the offset is 3, because the leading 1-bit is removed afterwards. When writing the length=3 in gamma code it becomes:10,1
6. for delta code, the leading 1-bit of the offset is removed now, generating an offset of 10
7. The code of 5 in delta is 10101

Constructing Index_Version_2.compressed:
------------------------------------------
For dictionary compression front coding is used and for the posting file delta encoding for the gaps between document-ids is used.

Front coding:
Ex: 8automat*a1?e2?ic3?ion

Delta encoding:
To write 9 in gamma and delta codes we perform the following operations:
1. write 9 in binary as 1001
2. for gamma code, remove the leading 1-bit to obtain the offset: 001
3. The length of the offset is 3:
• In unary the length is 1110
4. The code of 9 in gamma is 1110001 (or 1110,001 – to represent [length, offset])
5. for the delta code, the length of the binary representation is 4
6. The length is represented in gamma code: 11000
7. The leading 1-bit of the binary representation is removed, generating the offset 001
8. The code of 9 in delta is 11000001
